# CODING_RULES

## TypeScript

- modo `strict`;
- evitar `any`;
- usar tipos explícitos nas fronteiras;
- preferir uniões discriminadas;
- validar dados externos;
- funções pequenas e com responsabilidade única.

## Código

- nomes claros;
- sem duplicação desnecessária;
- sem ficheiros genéricos de utilitários;
- sem comentários que apenas repetem o código;
- comentários apenas para contexto e decisões;
- erros esperados devem usar resultados tipados.

## Imports

- evitar ciclos;
- usar aliases definidos pelo projeto;
- ordenar imports;
- não importar infraestrutura dentro do domínio.
