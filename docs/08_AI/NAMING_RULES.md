# NAMING_RULES

## Ficheiros

- componentes React: `PascalCase.tsx`;
- hooks: `useSomething.ts`;
- casos de utilização: `VerbNoun.ts`;
- entidades: `PascalCase.ts`;
- testes: `*.test.ts` ou `*.spec.ts`;
- constantes: nomes claros, sem abreviações obscuras.

## Exemplos

```text
CreateChildProfile.ts
SubmitExerciseAnswer.ts
MathGameScreen.tsx
useActiveProfile.ts
ExerciseRepository.ts
```

## Regras

- usar inglês no código;
- usar português de Portugal no conteúdo visível inicial;
- não usar nomes como `data`, `thing`, `manager` sem contexto;
- evitar abreviaturas não documentadas.
