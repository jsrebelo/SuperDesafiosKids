# DEVELOPMENT_WORKFLOW

## Fluxo por tarefa

1. Selecionar uma tarefa pronta.
2. Confirmar requisitos e critérios de aceitação.
3. Criar branch.
4. Implementar alteração pequena.
5. Adicionar testes.
6. Executar lint, typecheck e testes.
7. Rever acessibilidade e offline.
8. Atualizar documentação.
9. Criar pull request.
10. Rever e integrar.

## Nome de branch

```text
feature/<descricao-curta>
fix/<descricao-curta>
docs/<descricao-curta>
refactor/<descricao-curta>
```

## Commits

Usar Conventional Commits:

```text
feat(profiles): add child profile creation
fix(math): prevent duplicate answer submission
docs(ai): add Codex workflow
test(learning): cover difficulty reduction
```
