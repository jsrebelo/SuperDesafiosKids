# COMPONENT_RULES

## Componentes React

Devem:

- ter responsabilidade visual clara;
- receber dados por propriedades;
- emitir eventos por callbacks;
- usar tokens do Design System;
- incluir estados de loading, vazio e erro quando aplicável;
- suportar teclado e leitores de ecrã;
- evitar regras de domínio.

## Limites

- dividir componentes excessivamente grandes;
- extrair lógica reutilizável para hooks ou casos de utilização;
- não criar abstrações prematuras;
- não usar Phaser para menus, formulários ou área dos pais.
