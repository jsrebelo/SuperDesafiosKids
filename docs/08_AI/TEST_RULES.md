# TEST_RULES

## Obrigatório

- regras do ALE: testes unitários;
- geradores: testes determinísticos;
- casos de utilização: testes de integração;
- componentes críticos: testes de interface;
- fluxos principais: testes end-to-end.

## Regras

- testes devem descrever comportamento;
- não testar detalhes internos sem necessidade;
- usar repositórios em memória;
- controlar relógio e aleatoriedade;
- não depender de rede;
- não tornar testes menos rigorosos para fazer passar uma alteração.
