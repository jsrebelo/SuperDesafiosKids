# Release Candidate Checklist

## Código

- [ ] TypeScript strict aprovado.
- [ ] Build de produção aprovado.
- [ ] Sem erros de lint.
- [ ] Testes unitários aprovados.
- [ ] Testes de integração aprovados.
- [ ] Testes end-to-end aprovados.

## Produto

- [ ] Criar e selecionar perfil.
- [ ] Matemática funcional.
- [ ] Palavras funcional.
- [ ] ALE funcional.
- [ ] Recompensas funcionais.
- [ ] Área dos pais protegida.
- [ ] Estatísticas corretas.
- [ ] Limite diário funcional.
- [ ] Importação e exportação testadas.
- [ ] Relatório testado.

## Qualidade

- [ ] Funciona offline.
- [ ] PWA instalável.
- [ ] Alto contraste.
- [ ] Movimento reduzido.
- [ ] Fonte amigável para dislexia.
- [ ] Navegação por teclado.
- [ ] Sem mensagens negativas.
- [ ] Sem publicidade.
- [ ] Sem recolha desnecessária.

## Dados

- [ ] Migração localStorage → IndexedDB validada.
- [ ] Importação não duplica dados de forma insegura.
- [ ] Exportação inclui versão de esquema.
- [ ] Dados antigos permanecem recuperáveis.
- [ ] Não existem nomes em logs.

## Release

- [ ] Versão atualizada.
- [ ] Changelog atualizado.
- [ ] Notas de release.
- [ ] Build de preview.
- [ ] Plano de rollback.
- [ ] Aprovação final.
