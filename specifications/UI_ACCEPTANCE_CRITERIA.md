# UI_ACCEPTANCE_CRITERIA

- [ ] Um objetivo principal por ecrã.
- [ ] Botões principais têm área de toque adequada.
- [ ] A criança chega ao início em até duas ações.
- [ ] A aplicação não depende de hover.
- [ ] Todos os estados usam mais do que apenas cor.
- [ ] A interface suporta alto contraste.
- [ ] A interface suporta movimento reduzido.
- [ ] Textos visíveis usam internacionalização.
- [ ] A área infantil não apresenta ações destrutivas.
- [ ] Feedback de resposta é imediato.
- [ ] Nenhuma mensagem usa “Errado”, “Perdeste” ou equivalente.
- [ ] Componentes principais têm foco visível.
- [ ] Jogos em canvas mantêm ações essenciais em HTML.
- [ ] Música e efeitos podem ser desligados separadamente.
- [ ] O layout funciona em telemóvel, tablet e desktop.
